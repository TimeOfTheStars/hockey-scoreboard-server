# Hockey Scoreboard Server

Локальное Tauri-приложение: HTTP API в формате контракта внешнего источника для **Hockey Desktop Host** и веб-редактор всех полей `GameState`.

## Запуск в разработке

```bash
npm install
npm run tauri dev
```

В верхней части окна показан URL вида `http://127.0.0.1:8765/api/vmix` (порт может отличаться, если `8765` занят) — его нужно вставить в поле «URL внешнего API» в Host.

HTTP-сервер слушает **все интерфейсы** (`0.0.0.0`), чтобы можно было открыть панель с телефона по Wi‑Fi.

## Панель в браузере / с телефона

1. Соберите фронт: `npm run build`.
2. Запустите приложение (`npm run tauri dev` или собранный бинарник) — Axum отдаёт `dist` с того же порта.
3. На телефоне откройте **`http://<IP-ПК>:<порт>/mobile`** — только пульт (крупные кнопки). Полная панель: `http://<IP>:<порт>/`.

На пульте **«Сброс»** выставляет стандартное время периода `20:00` и останавливает таймер; текущее время до старта задаётся одним полем **Timer** в полной панели.

В режиме только Vite (`localhost:1420`) прокси перенаправляет `/api` на `127.0.0.1:8765`, маршруты вроде `/mobile` отдаются как SPA.

## Сборка

```bash
npm run tauri build
```

При **Старт** (`Running: true`) встроенная задача раз в секунду уменьшает строку `Timer` (`MM:SS`) и при `00:00` сама ставит `Running: false`. При активном большинстве параллельно тикает `PowerPlayTimer`. Формат времени должен быть двухчастным, напр. `20:00` или `03:42`.

## HTTP API

| Метод | Путь | Назначение |
|--------|------|------------|
| `GET` | `/api/vmix` | То же, что опрашивает Host (~каждые 800 мс) |
| `GET` | `/api/editor/state` | Текущее состояние (для отладки или внешних скриптов) |
| `PATCH` | `/api/editor/state` | Частичное обновление JSON (поверхностный merge) |
| `POST` | `/api/editor/reset` | Сброс состояния к значениям по умолчанию |

Проверка из терминала:

```bash
curl -sS "http://127.0.0.1:8765/api/vmix"
```

## Технологии

- Tauri 2, Rust (Axum + Tokio), React + Vite + TypeScript.
